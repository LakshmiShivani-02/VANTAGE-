# Contributing to VANTAGE

First off, thank you for considering contributing to VANTAGE! It's people like you that make VANTAGE such a powerful tool for the global research community.

## Code of Conduct
By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

## How Can I Contribute?

### Reporting Bugs
* Check the existing issues to ensure the bug hasn't already been reported.
* Use the **Bug Report** template provided in GitHub issues.
* Provide a clear summary, steps to reproduce, and any relevant logs or screenshots.

### Suggesting Enhancements
* Open a new issue with a clear title and description.
* Explain why this enhancement would be useful to most VANTAGE users.

### Pull Requests
1. Fork the repository and create your branch from `main`.
2. Ensure your code follows the existing style and is well-documented.
3. If you've added code that should be tested, add tests.
4. Ensure the test suite passes.
5. Provide a detailed summary of your changes in the PR template.

## Style Guide
- **Python**: Follow PEP 8. Use type hints where appropriate.
- **Javascript/React**: Use functional components and hooks. Follow the established ESLint configuration.
- **Documentation**: Use clear, concise language. Markdown is preferred for all docs.

## Research Contributions
We highly encourage contributions related to:
- New agentic reasoning patterns.
- Integration with additional global datasets.
- Benchmarking improvements.
- Relational graph synthesis optimizations.

## Getting Help
If you have questions, feel free to open a discussion on GitHub or reach out to the core maintainers.
